# panel-nacup26-resource

## 项目简介

本仓库用于归档与发布多个独立的 HTML 页面。这些页面是独立的资源文件，不针对任何单一域名或特定网站，旨在以静态形式提供可访问的页面内容。

## 目录说明

```
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面所需的样式、脚本、图片等静态资源
├── index.html      # 仓库根目录下的入口页面（如有）
└── README.md       # 本文件
```

- `pages/`：每个子目录或文件代表一个独立页面，命名清晰以区分不同用途。
- `assets/`：按页面或功能分类，存放对应的 CSS、JavaScript、图像等文件。

## 页面归档说明

- 所有 HTML 页面均以独立、自包含的方式存放，便于直接通过浏览器打开或部署到任意静态托管服务。
- 页面之间无强制依赖关系，可单独使用。
- 归档内容可能包含示例、工具页面、信息展示页等，具体用途详见对应页面内的注释或说明。

## 维护说明

- 本仓库定期更新，新增或修正页面内容。
- 如需提交新页面或修改已有页面，请遵循以下原则：
  - 页面文件应放置在 `pages/` 下合适的子目录中。
  - 资源文件统一放入 `assets/`，避免在页面目录内重复存放。
  - 保持页面独立性，不要引入外部不可控资源。
- 提交前请确保页面在主流浏览器中能正常渲染。

## 使用方式

1. 克隆或下载本仓库。
2. 直接在浏览器中打开 `pages/` 下的任意 HTML 文件。
3. 或部署到任何 Web 服务器 / 静态站点托管平台。
